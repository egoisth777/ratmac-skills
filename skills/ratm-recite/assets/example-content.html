The ratmac skill set is 11 composable skills that automate the scheduler data model (proj / slice / task triplet). The scheduler defines the shape; ratmac is the automation that maintains it.

The 11 skills

Stateless / no-write

skillrole
ratmac-initLoads invariants R1–R18 + the output-contract template. Pulled in first by every other skill.
ratmac-routeSession boot. Reads active proj state, lists slice + tasks, classifies a suggested next-action mode.
ratmac-lintRead-only checker: schema + invariants + dangling links. --strict adds spec-vs-disk audit.

Write skills

skillroleregen?
ratmac-kickoffScaffold a tier (proj/slice/task) + parent state/log rows.no (run lint yourself)
ratmac-checkpointWork-pause snapshot: bump state, append log, add to ## affects.no
ratmac-mutateIn-place plan/approach/ticket revision.no
ratmac-scopesole/dual scope expand/contract + scope-history.yes
ratmac-closeTask done/abandoned, freeze affects, mv to archive.yes (regen only)
ratmac-transitSlice/proj transition: summary, status, mv to archive.yes + lint

Generated + orchestrator

skillrole
ratmac-regenRebuild GENERATED-only content (residuals, fenced rollups). Byte-idempotent → drift detector.
ratmac-autoOrchestrator. State machine + closed-loop; delegates to the others; stops on ambiguity.

Orchestration (ratmac-auto)

INIT → CLASSIFY → EVIDENCE → ROUTE → EXECUTE → VERIFY → REPORT
                              |                                  |
                              +--> STOP (HUMAN_DECISION_REQUIRED|BLOCKED)--+
Closed-loop: every routed branch ends in completed-in-scope, blocked, human-decision-required, or handoff-emitted. On any ambiguity, STOP and report — never guess the route.
